Cardio

Short Description

Cardio is a health application designed to aid users in tracking their blood pressure and heart rate using a bluetooth connected device. Patients receive daily tasks they need to complete; They might have to complete health questionnaires or measure their blood pressure and heart rate.
The app is still in development.

My Role

A big part of this app was already done. My initial task was to integrate the Objective C library used to connect the bluetooth connected health device. Afterwards I also integrated a database to store user's data. Displaying the data on a period of time was also a challenge I tackled by building a custom charts class. Because this was a medical app I had to get accustomed to the FHIR specifications.

MasjidLink

Short Description

MasjidLink is an app designed for Muslims to connect with the Mosques or Businesses. Users can see all the events and announcements of an Organization (either Mosque or Business) after linking to it and they'll also get Adhan and Iqamah time changes notifications. Admins can also do quick edits of their Organizations from within the app.
The app is currently on the AppStore and being used on a daily basis.

My Role

Developer in a team of two. I created the basic structure, defined the architecture and laid out the foundation of the app. I've also integrated CI using Fastlane. Afterwards, my role shifted towards supervising and I only stepped in when needed.

Technologies

UIKit, RxSwift, Continuous Integration, Storyboards + Programmatic User Interface, Networking, Firebase, Push Notifications

Keyvent

Short Description

Keyvent is an online ticketing platform. The app enables users to see featured events near them, search for events using different filters, buy tickets and see them in app. When a ticket is bought, a QR-code is generated and users have to get it scanned at entrance.
After a few months on the AppStore, I had to take it down since it lost backend support.

My Role

Sole developer. There was nothing created, only some vague mock-ups. Going through all the steps from defining the features, to creating the designs and finally the application was a great journey. We wanted to release the app in China as well, that's why I had to create multiple targets and some classes were totally different for the Chinese version of the app. One big difference was that we couldn't use Stripe to handle payments in China. We also couldn't use Google and Facebook logins.

Technologies

Design, Viper, Networking Alamofire, CoreData, Social Logins, Multiple Targets and Schemes management, Stripe, Alipay, Deep Linking, Universal Links

Keyvent Manager

Short Description

This app allows event employees to scan event tickets (QR-codes) at the entrance. Employees can grant or deny access. There's also a fallback to enter the code manually if the scan doesn't work.
This application is currently on the AppStore and it's being used to scan tickets.

My Role

Sole developer. Same as Keyvent app, I ran through the whole process from design to development with this app. Because this app uses camera to scan QR-codes, battery can easily drain so the scanning stops after a few minutes of inactivity. We also wanted to improve the scanning experience so I've added some beautiful animations using Lottie.

Technologies

UIKit, Lottie Animations, Networking using URLSession, CoreData, AVFoundation, Crashlitycs

Noise Festival

Short Description

This was a companion app for a festival that took place in Brasov, Romania called Noise Festival. Users were able to see the artists, listen to their songs, see the news, the partners, info about the city and how to get there and most importantly, the stage times. Technologies

UIKit, Firebase, Push Notifications, Custom Animations
